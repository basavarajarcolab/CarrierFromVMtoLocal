The system shall enforce password policies based on the Active Directory (AD) configuration when users authenticate via LDAP or Single Sign-On (SSO). This ensures that password rules (such as complexity, expiration, and history) are consistent with the organization's centralized identity management policies and reduces the need for duplicate policy management within the application.

Acceptance Criteria

For users logging in via LDAP or SSO, the system shall retrieve and enforce the password policy defined in the integrated Active Directory (AD).

The password policy enforced shall include, but is not limited to:

Minimum and maximum password length

Complexity rules (e.g., uppercase, lowercase, numeric, special characters)

Password expiration time

Password reuse restrictions (history)

The system shall not allow local override of the AD password policy for LDAP/SSO users.

The system shall display meaningful error messages when a user attempts to change their password in a way that violates the AD policy.
